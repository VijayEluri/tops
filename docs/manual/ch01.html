Who is this manual for?

There are two main reasons for this manual. One is to provide users of the TOPS system with a reference; the other is to allow programmers to re-use or alter parts of the system.

What does the manual contain?

Each major part of the system is documented in a separate chapter. Since the parts are necessarily integrated into a whole, there is a final chapter on integration.

In addition, there are a couple of appendices on building the code - and this manual! - from source. Each part of the system is a task in a buildfile.

What software do I need?

To run most of the Java classes, nothing more than a version of Java is required. Where particular programs need more than this it is indicated in the appropriate chapter.

To build the software from source, the java make-equivalent software called ant is required. To make a version of this manual, some more software (DocBook and an XSL processor) is required. This is described in more detail in the appendices.
